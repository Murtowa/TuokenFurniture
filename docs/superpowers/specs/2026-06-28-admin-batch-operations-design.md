# 管理后台批量操作设计

## 概述

为后台管理的用户、商品、分类三个模块添加批量操作功能。采用表格多选 + 顶部操作栏模式。

## 批量操作清单

### 用户管理
- 批量启用（status: 1）
- 批量禁用（status: 0）
- 不做批量删除（当前连单个删除也没有，保持一致）

### 商品管理
- 批量上架（status: 1）
- 批量下架（status: 0）
- 批量删除
- 批量移动分类

### 分类管理
- 批量删除（带保护检查）

## UI 方案

- 每个管理页 el-table 加 `type="selection"` 列 + 表头全选复选框
- 选中 ≥1 项时表格上方浮现操作栏，取消全部选中后隐藏
- 操作栏文案：`当前页已选 N 项` + 操作按钮
- 翻页/搜索/筛选时清空选中（跨页选择不保留，避免选中丢失的认知偏差）
- 批量删除操作二次确认（ElMessageBox.confirm）
- 批量移动分类弹出分类选择弹窗

### 操作栏布局

```
用户:   当前页已选 N 项  [批量启用] [批量禁用]
商品:   当前页已选 N 项  [批量上架] [批量下架] [批量删除] [移动分类▾]
分类:   当前页已选 N 项  [批量删除]
```

## 后端 API

所有批量路由挂载在 `/api/admin/` 下，复用 `auth + adminAuth` 中间件。

| 方法 | 路由 | 请求体 | 说明 |
|------|------|--------|------|
| PUT | `/users/batch-status` | `{ ids, status }` | 批量启/禁用用户 |
| PUT | `/products/batch-status` | `{ ids, status }` | 批量上/下架商品 |
| PUT | `/products/batch-category` | `{ ids, categoryId }` | 批量移动分类 |
| POST | `/products/batch-delete` | `{ ids }` | 批量删除商品 |
| POST | `/categories/batch-delete` | `{ ids }` | 批量删除分类（带保护检查） |

### 返回格式

批量操作返回实际影响行数，前端据此给出精确反馈：

```json
{ "code": 0, "message": "已上架 2 个商品，3 个商品无需变更", "data": { "affected": 2, "skipped": 3, "total": 5 } }
```

### 实现要点

- 使用 `WHERE id IN (?)` 单条 SQL，不做逐个循环
- 分类批量删除前检查：是否有关联商品或未删除的子分类。有则拒绝并返回具体哪些分类受阻及原因
- 状态类操作（启/禁用、上/下架）：静默跳过已匹配目标状态的行，返回实际变更数

## 不做

- 用户批量删除
- 操作日志
- 分批/任务队列（当前数据规模单次 SQL 足够）

## 前置路由注册

注意：Express 路由 `/users/batch-status` 需在 `/users/:id/status` 之前注册，否则 `batch` 会被 `:id` 匹配。
